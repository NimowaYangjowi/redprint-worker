## Key Interactions to Verify
- BackupScheduler correctly calculates ms until next backup hour
- BackupScheduler skips if today's backup already exists in R2
- pg_dump child process execution and exit code handling
- gzip -t integrity verification catches corrupt files
- Buffer-based R2 upload succeeds and handles failures
- Retention deletes backups older than 7 days, keeps minimum 1
- Unified error handler catches all error types and logs with [BACKUP] prefix
- Graceful shutdown waits for running backup (30s timeout)

## Edge Cases
- Empty DB (0 rows) produces valid small backup
- Concurrent trigger prevention (setTimeout fires only once)
- Docker restart mid-backup: R2 check prevents duplicate
- Retention with exactly 1 backup: must NOT delete
- Retention with 0 backups: no-op
- File size = 0 after pg_dump: reject before upload
- pg_dump binary not found (ENOENT): graceful error
- Disk full during dump (ENOSPC): cleanup partial file

## Critical Paths
- Full backup flow: pg_dump → gzip → verify → upload → retention (E2E with mocks)
- Error recovery: any step failure → cleanup + skip + schedule next day
